- `Property/web/src/lib/dividendTax.ts:10-12` hardcodes `DIV_BASIC=0.0875`, `DIV_HIGHER=0.3375`. FA 2026 s.4 raised these to **10.75% / 35.75%** from 6 Apr 2026 (additional 39.35% is unchanged & correct).

- Same stale rates also appear as **user-facing content** on `/property-tax-rates` (`page.tsx:275`, labelled "Verified figures") and the incorporation blog page.
- **Fix:** set the two constants + update labels/FAQ/explainer in `dividend-tax-calculator.ts` and `premium/tools/incorporation.ts:279`; sweep the two content pages.

### C2 — Dividend golden test pins the WRONG figure (CODE — inverts the safety net)
- `tests/calculator-goldens.test.ts:443-445` asserts `'£1,706'` (the stale value). The suite's own header calls itself the "STOP gate" for stale figures, yet it omits dividend *rates* from its sentinel list — which is exactly how C1 slipped through. Any correct fix to C1 will read as a "regression."
- **Fix:** after C1, assert £2,096 and add `DIV_BASIC===0.1075 / DIV_HIGHER===0.3575` sentinels.

## Dashboard — full answer to "what are the JS errors and why?"

**They are React hydration-mismatch errors, not crashes.** The console is structurally sound (verified): Next 15 promise-`params` are awaited everywhere, the recharts SSR pattern (`initialDimension` + `'use client'` leaf charts + serialisable props) is correct for recharts 3.8.0, the server data layer degrades to `[]`/zero-filled on missing env or empty rows (no null-access crashes), list keys are unique, auth is cookie-based + timing-safe, and the service-role key is strictly server-only.

The errors come from **client components formatting timestamps in the browser's local timezone**:
1. `TrendChart` chart ticks/tooltips (`toLocaleTimeString`/`toLocaleString`, no `timeZone`).
2. Visitor-timeline `clockTime`/Story header (`journey.ts`, no `timeZone`).

The page is SSR'd on Vercel (**UTC**) then hydrated in a UK browser (**BST** ~Apr–Oct). A 14:30 UTC bucket renders `14:30` on the server and `15:30` on the client, so React logs *"Text content does not match server-rendered HTML / Hydration failed"* and throws away + re-renders that subtree. Reproduced empirically; the existing test even encodes it (`/14:30|15:30/`). Lower-risk cousin: `VisitorsTable.tsx` relative `ago()` computed at render can drift across a bucket boundary between SSR and hydration.

**One fix clears all of it:** pass `timeZone:'UTC'` to every `toLocale*` call in `TrendChart.tsx` and `journey.ts` (it matches the "UTC buckets" header already shown), or render the timestamps after mount. `journey.ts` is shared, so fixing it there fixes every console that uses it.

## Systemic insight
Almost every wrong-number content finding is the same shape: a locked ground-truth change (SDLT 15→17, surcharge 3→5, dividend, WDA 18→14, mileage 45→55p, CGT date, Wales-in-2027) was applied to **most** pages but **missed a residual cohort**, leaving the corpus internally contradictory page-to-page. The calculator golden suite — the intended STOP gate — also omitted dividend *rates* from its sentinels. **The single highest-leverage fix is an estate-wide "stale-figure sweep gate"** (grep sentinels in CI + golden sentinels for every locked rate) so the next ground-truth change can't half-land.
