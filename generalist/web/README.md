# Dental Finance Partners — Next.js Site

Production-ready UK dental practice finance & accounting site. Mobile-first, SEO-optimized, integrated with Google Apps Script lead form and Python blog pipeline.

---

## Quick Start

### Development

```bash
cd web
npm install
npm run dev
```

Site runs at `http://localhost:3000`.

### Production Build

```bash
npm run build
npm start
```

---

## Environment Variables

Create `web/.env.local`:

```bash
# Production site URL (no trailing slash)
NEXT_PUBLIC_SITE_URL=https://dentalfinancepartners.co.uk

# Google Apps Script web app URL (for lead form submissions)
NEXT_PUBLIC_GOOGLE_APPS_SCRIPT_URL=https://script.google.com/macros/s/YOUR_DEPLOYMENT_ID/exec
```

See `../GOOGLE_APPS_SCRIPT_SETUP.md` for Apps Script setup.

## Key Features

### SEO
- **Meta tags** — Title templates, descriptions, OG tags on every page.
- **Canonical URLs** — Set on all pages.
- **Sitemap** — Auto-generated at `/sitemap.xml` (includes all routes + blog posts).
- **Robots.txt** — `/robots.txt` (allows all except `/thank-you`).
- **JSON-LD Schema** — Organization schema on homepage; BlogPosting + FAQPage on blog posts.
- **Semantic HTML** — Proper heading hierarchy, `<article>`, `<section>`, `<time>`, `lang="en-GB"`.

### Blog System
- **Markdown source** — Blog posts live in `content/blog/*.md` (generated by Python pipeline in `../BA NEXT JS SITE`).
- **Front matter** — `title`, `slug`, `date`, `author`, `category`, `metaTitle`, `metaDescription`, `h1`, `summary`, `image`, `altText`, `schema` (optional JSON-LD override), `faqs` (array).
- **Auto-routing** — Drop a `.md` file in `content/blog/` and it appears at `/blog/{slug}`.
- **Related posts** — Shown at bottom of each article (by category).

### Lead Form
- **Fields** — Name, email, phone, role (associate/owner/group), optional message.
- **Submission** — Posts JSON to `NEXT_PUBLIC_GOOGLE_APPS_SCRIPT_URL` (Google Apps Script → Google Sheet).
- **Validation** — Client-side checks; UK phone format; email format.
- **Redirect** — Success → `/thank-you` (or inline message if `redirectOnSuccess={false}`).

### Design System
- **Fonts** — Plus Jakarta Sans (body), Cormorant Garamond (serif headings).
- **Colors** — Navy (`#001B3D`), gold (`#B8975D`), light neutrals.
- **Components** — `card-premium`, `card-flat`, `hero-glass`, `section-label`, button variants.
- **Motion** — Subtle hero animations (respects `prefers-reduced-motion`).

## Content Management

### Adding a Blog Post Manually

1. Create `content/blog/your-slug.md`:

```markdown
---
title: "Your Article Title"
slug: "your-slug"
date: "2026-03-27"
author: "Dental Finance Partners"
category: "Associate tax"
metaTitle: "SEO title (50-60 chars)"
metaDescription: "SEO description (140-160 chars)"
h1: "Page heading"
summary: "Short summary for cards"
altText: "Image alt text"
image: ""
schema: ""
faqs:
  - question: "FAQ question?"
    answer: "FAQ answer."
---

<h2>Your first section</h2>
<p>Your content in HTML format...</p>
```

2. Restart dev server or rebuild — post appears at `/blog/your-slug`.

## Maintenance

### Updating Site Config

Edit `src/config/site.ts` to change:
- Site name, tagline, description
- Navigation links
- Footer links
- Location pages
- Contact details

### Updating Styles

- **Global styles** — `src/app/globals.css`
- **Color tokens** — CSS variables in `:root` block
- **Component styles** — Tailwind utilities in components
- **Layout primitives** — `src/components/ui/layout-utils.ts`

### Adding a New Page

1. Create `src/app/your-page/page.tsx`.
2. Add to `src/app/sitemap.ts` static paths.
3. Optionally add to `siteConfig.nav` or `siteConfig.footer`.

## Troubleshooting

### "Unstyled HTML" / No CSS

**Cause:** Tailwind v4 scanned the wrong directory (Next.js inferred `C:\Users\user` as root due to extra `package-lock.json`).

**Fix:** Already applied — `globals.css` uses `@import "tailwindcss" source("..")` and `next.config.ts` sets `outputFileTracingRoot`.

**Verify:** Run `npm run dev` from the `web` folder, hard refresh browser (Ctrl+Shift+R).

### Form Not Submitting

- Check `NEXT_PUBLIC_GOOGLE_APPS_SCRIPT_URL` is set in `.env.local`.
- Restart dev server after adding env vars.
- Check Apps Script is deployed as **Anyone** access.

### Blog Post Not Appearing

- Check file exists: `content/blog/{slug}.md`
- Check front matter has required fields: `title`, `slug`, `date`, `h1`, `summary`
- Restart dev server

### Build Fails

- Delete `.next` folder: `Remove-Item -Recurse -Force .next`
- Run `npm run build` again
