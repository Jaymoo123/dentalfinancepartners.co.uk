# Dental Finance Partners - Project Summary

Complete overview of the lead generation website built for UK dental accounting services.

---

## Project Goal

Build a **production-ready lead generation website** for Dental Finance Partners (`dentalfinancepartners.co.uk`) — a UK accounting firm specializing exclusively in dental practices.

**Target Audience:**
- Associate dentists (tax, expenses, self-assessment)
- Practice owners (accounts, VAT, profit extraction)
- Multi-practice groups (acquisitions, financial planning)

**Primary Objectives:**
1. Generate qualified leads through contact forms
2. Establish authority through SEO-optimized blog content
3. Provide clear service positioning (dental-only specialist)
4. Mobile-first, professional design
5. Easy content management and lead tracking

**Content Management:**
- Markdown files in `web/content/blog/`

**Blog Generation Pipeline:**
- **Tool:** Python + Anthropic Claude Sonnet 4
- **Topics:** Stored in Supabase `blog_topics` table
- **Script:** `BA NEXT JS SITE/generate_blog_supabase.py`
- **Output:** Production-ready Markdown with SEO metadata
- **Cost:** ~$0.03 per 1,500-2,500 word post

**Workflow:**
1. Add topics to Supabase table
2. Run `python generate_blog_supabase.py`
3. Script generates full blog post and saves to `web/content/blog/`
4. Commit and deploy to Vercel
5. Post appears at `/blog/{slug}` automatically

## Technical Decisions Made

### Why Supabase Instead of Google Sheets?
- **CORS issues:** Google Apps Script blocks cross-origin requests
- **Reliability:** Production-grade PostgreSQL vs. spreadsheet
- **Developer experience:** REST API, real-time updates, better error handling
- **Scalability:** Can add email notifications, webhooks, admin dashboards

### Why Anthropic Claude Instead of OpenAI?
- **You provided the API key:** Already had Anthropic access
- **Quality:** Claude Sonnet 4 produces high-quality, structured content
- **Cost:** Competitive pricing (~$0.03 per blog post)
- **Reliability:** Consistent output format for parsing

### Why Next.js Instead of WordPress/Webflow?
- **Performance:** Static generation = fast page loads
- **Developer control:** Full customization, no plugin limitations
- **Modern stack:** TypeScript, React, Tailwind
- **Free hosting:** Vercel free tier is generous
- **SEO-friendly:** Built-in sitemap, metadata, schema support

### Why Markdown for Blog Posts?
- **Version control:** Blog content tracked in Git
- **Portability:** Easy to migrate to other platforms
- **Developer-friendly:** Simple format, no database needed
- **Fast:** No CMS queries, just read files at build time

## Challenges Solved

### 1. **Unstyled Pages (Tailwind Not Working)**
**Problem:** Site rendering as plain HTML, no styles  
**Root Cause:** Next.js inferring wrong project root due to extra `package-lock.json` in parent directory  
**Solution:** Added `outputFileTracingRoot` to `next.config.ts` and `@import "tailwindcss" source("..")` to `globals.css`

### 2. **Missing Logo (Blank White Box)**
**Problem:** `next/image` trying to load non-existent `/brand/logo.png`  
**Solution:** Created typographic text-based logo components (`BrandLogoHero`, `BrandWordmarkHomeLink`)

### 3. **Google Apps Script CORS Errors**
**Problem:** Form submissions failing with "Failed to fetch"  
**Root Cause:** Google Apps Script doesn't support CORS properly for cross-origin requests  
**Solution:** Switched to Supabase REST API with proper CORS headers

### 4. **Phone Validation Too Strict**
**Problem:** UK mobile numbers being rejected as invalid  
**Solution:** Refined regex and split validation into two checks (valid characters + minimum digits) with specific error messages

### 5. **Build Cache Corruption**
**Problem:** Dev server crashing with webpack chunk errors  
**Solution:** Killed stale processes and deleted `.next` directory to force clean rebuild

### 6. **Port Already in Use**
**Problem:** `EADDRINUSE: address already in use :::3000`  
**Solution:** Used `netstat` to find PID, killed process with `Stop-Process`

### 7. **GitHub Secret Scanning**
**Problem:** Push rejected due to API keys in commit history  
**Solution:** Moved API keys to environment variables, redacted from config files
