## STAGE 3: Content Generation (Python + Claude)

### Purpose
Generate blog content from topics using Claude API.

### Scripts

#### Single Post Generator
**File**: `Property/generate_blog_supabase.py`, `Dentists/generate_blog_supabase.py`

**Process**:
1. Fetch ONE unused topic from Supabase (priority-based)
2. Call Claude Sonnet 4 API with topic + SEO guidance
3. Parse Claude's response (structured format with `==FIELD==` delimiters)
4. Generate markdown file with frontmatter
5. Mark topic as `used=true` in Supabase

**Usage**: Run once per post
```bash
python Property/generate_blog_supabase.py  # Generates 1 post
```

#### Batch Generator
**File**: `Property/generate_all_automated.py`, `Dentists/generate_all_automated.py`

**Process**:
1. Count unused topics in Supabase
2. Loop up to 20 times
3. Call single generator as subprocess
4. Wait 2 seconds between calls

**Usage**: Run once for up to 20 posts
```bash
cd Property
python generate_all_automated.py  # Generates up to 20 posts
```

#### Phase-Specific Generator
**File**: `Property/generate_phase1.py`, `Dentists/generate_phase1.py`

**Process**:
1. Loop fixed number of times (12 for Property, 9 for Dentists)
2. Call single generator as subprocess
3. Display detailed progress

**Usage**: Run once per phase
```bash
cd Property
python generate_phase1.py  # Generates exactly 12 posts
```

### Configuration
**File**: `Property/config_supabase.py`, `Dentists/config_supabase.py`

**Imports from**: `shared_supabase_config.py`

**Contains**:
- API keys (Anthropic, Supabase)
- Output directories
- Site URLs
- Blog system prompts
- Categories
- Internal links

### Claude API Details
- **Model**: claude-sonnet-4-20250514
- **Max tokens**: 4096
- **Cost**: ~$0.18 per post
- **Caching**: Ephemeral cache for system prompt
- **Timeout**: 120 seconds per generation

### Integration Points
- **Input**: Unused topics from Supabase
- **Output**: Markdown files + Supabase updates
- **Next stage**: Next.js reads markdown files

## PRIORITY-BASED GENERATION CONTROL

### How It Works

**Generation scripts query Supabase**:
```sql
SELECT * FROM blog_topics_property
WHERE used = false
ORDER BY publish_priority DESC NULLSLAST,
         keyword_difficulty ASC NULLSLAST,
         created_at ASC
LIMIT 1
```

**This means**:
1. Highest `publish_priority` first
2. Then lowest `keyword_difficulty` (easier topics)
3. Then oldest `created_at` (FIFO)

### Phase Control Method

**Phase 1 topics**: Set `publish_priority=10`  
**Phase 2 topics**: Set `publish_priority=9` (when imported)  
**Pre-existing topics**: Set `publish_priority=1`

**Result**: Phase 1 generates first, then Phase 2, then pre-existing.

### SQL Updates Used Tonight
```sql
-- Property Phase 1
UPDATE blog_topics_property 
SET publish_priority = 10 
WHERE slug IN ('what-does-a-property-accountant-do', ...);

-- Property pre-existing
UPDATE blog_topics_property 
SET publish_priority = 1 
WHERE slug IS NULL AND used = false;

-- Dentists Phase 1
UPDATE blog_topics 
SET priority = 10 
WHERE slug IN ('associate-dentist-tax-deductions-2026', ...);

-- Dentists pre-existing
UPDATE blog_topics 
SET priority = 1 
WHERE slug IS NULL AND used = false;
```
